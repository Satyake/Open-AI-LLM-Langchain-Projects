# Open-AI-LLM-Langchain-Projects

An extensive collection of Open AI LLM-based projects and applications for Natural Language Processing (NLP) tasks and Generative AI use cases.

---

## Projects Overview:
### 1. **Hugging Face Transformer Fine-Tuning**
   - Supervised fine-tuning for various NLP tasks, such as:
     - Named Entity Recognition (NER)
     - Text Summarization
     - Neural Machine Translation (NMT)

### 2. **PDF Bot**
   - Uses Datastax Cassandra DB for vector search against user-generated queries.
   - Key Features:
     - User queries converted to text embeddings via LLM for similarity searches.
     - Hosted on Google Cloud NoSQL Cassandra DB.
     - Enables querying PDFs to retrieve relevant content.
     - Text embeddings for PDF data are precomputed for efficient search.

### 3. **Transformer for Text Summarization**
   - Fine-tuned Google/Pegasus model for text summarization use cases.

### 4. **BERT-Based Resume Classification System**
   - Implements BERT-based solutions for resume categorization and analysis.

### 5. **LangChain Use Cases**
   - A variety of practical use cases leveraging LangChain for building AI applications.

### 6. **Vector Search Notebooks**
   - Includes implementations for:
     - ChromaDB
     - PineCone
     - FAISS
     - RAG (Retrieval-Augmented Generation)
   - Focused on PDF-based vector search.

### 7. **Website Chatbot with RAG**
   - A chatbot powered by Retrieval-Augmented Generation (RAG).

### 8. **LoRA Fine-Tuning on LLaMA 2-7B**
   - Supervised fine-tuning for LLaMA 2-7B (Hugging Face Chat model).

### 9. **Telegram Chatbot with OpenAI**
   - Implementation of a Telegram chatbot using OpenAI and AIOGram.

### 10. **LlamaIndex for Mixtral**
   - Demonstrates how to implement LlamaIndex for Mixtral data.

### 11. **Fundamental RAG Implementations**
   - Example workflows showcasing RAG-based applications.

### 12. **Multimodal RAG System with LanceDB**
   - Designed for multimodal data processing, including:
     - Video-to-Image embeddings.
     - Video-to-Audio-to-Text embeddings stored in LanceDB.
   - Features:
     - Leverages GPT-4 Vision for query responses.
     - Augments results with a vector store (similarity searches within LanceDB).
